# 10 IO

## Goal
Use runtime IO modules with explicit effect declarations.

## Syntax you need
- `std.io` APIs require `effects { io }` and `capabilities { io }`.
- `std.fs` APIs require `effects { fs }` and `capabilities { fs }`.
- `std.net` APIs require `effects { net }` and `capabilities { net }`.
- `std.env` APIs require `effects { env }` and `capabilities { env }` (and cwd/home/temp helpers also require `fs`).
- `std.proc` APIs require `effects { proc, env }` and `capabilities { proc, env }` for launch/run operations.

## Runnable snippet
```aic
import std.io;

fn main() -> Int effects { io } capabilities { io } {
    eprint_str("[warn] ");
    eprint_int(7);
    eprint_str(" events");
    flush_stderr();

    println_str("stdout");
    print_bool(true);
    print_bool(false);
    println_bool(true);
    println_int(42);
    flush_stdout();
    0
}
```

## Run it
```bash
aic run examples/io/stderr_logging.aic
```

Expected visible output includes:
```text
[warn] 7 events
stdout
truefalsetrue
42
```

## What to remember
- IO is capability-gated by effects and matching capabilities, so declarations are part of the API contract.
- Most non-trivial IO calls return `Result[...]`; handle errors explicitly with `match` or `?`.
- For broader runtime coverage, see `examples/io/fs_all_ops.aic`, `examples/io/process_pipeline.aic`, and `examples/io/net_all_ops.aic`.
